# v4.2 — 범위 제한 위생 후보 라우팅과 OKF 시간 계약

## 문제

v4.1은 claim 노후도와 상태·링크 구조를 결정론적으로 검사했지만 어떤 문서를 의미 검토할지 제한하는 후보 계획이 없었다. 모든 문서를 모델 문맥에 넣을 수 없고, 기존 `timestamp`도 의미 변경·생성·검증·생명주기 시각을 구분하지 못했다.

## 변경

- 모델 호출 없는 전체 구조 스캔 뒤 최근·노후·위험 시드를 선정한다.
- 강한 유형 관계를 기본 최대 2-hop과 전체 노드 상한 안에서 확장한다.
- 극성·수치·명시적 반박 차이는 자동 판정이 아닌 검토 후보로 제한한다.
- 후보마다 선택 이유와 시드 경로를 남기고 같은 입력·시각에서 결정론적으로 재생한다.
- OKF `timestamp`를 마지막 의미 변경으로 고정하고 알려진 생성·검증·최신성·생명주기 시각을 별도 투영한다.
- 단순 render와 confidence 재계산은 검증 시각을 진전시키지 않는다.

## 마이그레이션

새 설정과 시간 필드는 추가형이다. 기존 문서의 `timestamp`는 유지하며 알려지지 않은 `created_at`을 추정하지 않는다. `content_updated_at`이 없는 이전 claim/source는 이미 기록된 생성·evidence·confidence 변화·평가·검색·생명주기 시각에서 보수적으로 의미 변경 시각을 계산한다.

## 롤백

`tools/wiki_hygiene.py`, `hygiene-plan` CLI, `hygiene_selection` 설정과 Skill 단계를 제거하면 v4.1 위생으로 돌아간다. 선택적 frontmatter 시간 확장은 이전 OKF 소비자가 무시할 수 있다. 원문, 사건 원장, claim/source 신뢰도와 생명주기는 변경하거나 삭제하지 않는다.

## 근거와 한계

구현 계약은 승인된 `RFC-6BD49C04ED49`와 `SPEC-LWS-001` v1.2.0을 따른다. 로컬 회귀와 릴리스 게이트 통과는 운영 환경의 의미 판단 품질을 인증하지 않으며 `production_certified=false`를 유지한다.
