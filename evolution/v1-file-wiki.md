# v1 — 영속 파일 위키

## 채택한 가설

매 질문마다 원문 조각을 다시 조합하는 대신, 불변 원문 위에 Agent가 유지하는 Markdown 종합문을 두면 연구가 세션 사이에 누적된다.

## 구조

```text
raw/ 원문 → 색인 → wiki/{sources,concepts,entities,comparisons} 지식 문서
                         ↘ 개요 / 종합 / 열린 질문
```

## 근거

- Karpathy의 LLM Wiki 아이디어 파일
- AI Engineer의 AI Research OS 발표와 공개 저장소

## 실험적 산출물

- 지정 Gist와 영상을 출처 페이지로 요약
- `index.md`, `overview.md`, `synthesis.md`, `open-questions.md`
- 원문·색인·위키의 점진적 읽기 순서

## 자체 린트 결과

1. 사용자 제공 자료의 관련성 1.0은 “반드시 읽어라”이지 신뢰도 1.0이 아니다.
2. 페이지 끝 출처 목록만으로는 문장별 증거 범위를 찾기 어렵다.
3. 개인화된 원자료가 확증편향을 강화할 수 있다.
4. 사람과 Agent의 기여·신원 스키마가 없다.
5. “살아 있음”은 요청 시 갱신일 뿐 주기적 연구나 품질 보장을 뜻하지 않는다.

## v2로 넘긴 요구사항

- 원자적 주장과 정확한 위치 식별자
- 출처 평가와 독립성 묶음
- 행위자 중립적 귀속과 검토
- 모순·오래됨·철회 상태
- 결정론적 검증기와 변조 흔적을 남기는 사건 로그
