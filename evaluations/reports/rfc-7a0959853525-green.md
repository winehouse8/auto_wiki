# RFC-7A0959853525 정책 전환 브리지 Green 근거

## 구현 결과

- 기준 commit `fd7271a36adff6f9118879aa2b435558c1358633`의 active `SPEC-GH-DELIVERY-001/v1.1.0`을 유지한 채, 사람이 검토한 정확한 `v1.1.0 → v1.2.0` `armed` edge와 승인·RFC를 설정에 추가했다.
- `begin --target-policy-version`은 branch 생성 전에 base Git 객체에서 from·to·base SHA·설정 blob SHA-256·승인·RFC proof를 만들고, branch 성공 뒤 v2 begin 영수증에 원자 기록한다.
- 활성화 `publish`는 설정 파일 하나에서 active 버전과 같은 edge 상태만 각각 target과 `consumed`로 바꾸는 diff를 허용하고 항상 draft 사람 검토로 끝낸다.
- 기존 일반 실행의 멱등성 key는 바꾸지 않고 전환 key에 target과 base 설정 digest를 추가했다.
- 사람이 활성화 PR을 병합한 뒤에는 결속된 prior review 영수증만 기존 PR 상태를 읽기 전용으로 재조회하고 실제 merge SHA를 기록한다. 자동화가 병합을 요청했다고 소급하지 않는다.

## 독립 감사 보강

- origin fetch·push URL은 각각 승인 저장소를 가리키는 정확한 한 개만 허용한다. 기준 fetch와 인증 push는 별칭이 아닌 승인된 HTTPS URL을 직접 사용한다.
- 모든 Git 판정과 push는 `refs/replace`를 무시해 같은 SHA 아래 다른 객체를 대입할 수 없게 했다.
- 정책 설정은 중복 key와 비표준 상수를 거부하고 JSON 값의 타입을 보존해 `true → 1` 변조를 차단한다.
- base 정책 digest는 text mode가 아니라 실제 Git blob bytes로 계산한다. CRLF fixture가 줄바꿈 정규화 없이 같은 SHA-256을 기록한다.
- 전환 여부와 무관하게 begin 영수증의 base를 publish 초기에 다시 확인한다. 이동한 과거 일반 영수증은 gate와 token loader 전에 차단한다.
- begin 영수증의 승인·RFC와 origin fetch·push URL도 context와 현재 저장소에 다시 결속한다.
