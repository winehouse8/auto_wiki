# 한국어 문서 정책

- 명세 ID: `SPEC-KO-DOCS-001`
- 버전: `1.2.0`
- 상태: 사용자 지시에 따라 승인된 설계 계약
- 상위 계약: [Living Wiki 하네스 UX와 전달 계약](harness-ux.md)

## 목표

사용자와 유지보수자가 읽는 Living Wiki·하네스 문서는 한국어를 기본 언어로 사용한다. 제목, 설명, 본문, 표 머리글, 상태 안내, 프롬프트, Skill UI 문자열과 자동 생성 문서의 고정 문구를 한국어로 제공한다. 기술적 추적성과 실행 호환성을 위해 보존해야 하는 값만 제한적으로 원문을 유지한다.

## 적용 범위

다음 파일의 사람이 읽는 문구에 적용한다.

- `wiki/**/*.md`의 수동 문서와 파생 문서
- `README.md`, 저장소 및 전역 `AGENTS.md`
- `governance/**/*.md`, `docs/**/*.md`, `evolution/**/*.md`, `research/**/*.md`, `prompts/**/*.md`
- `skills/**/SKILL.md`, Skill 참고 문서와 `agents/openai.yaml`의 사용자 표시 문자열
- `reports/**/*.md`, `evaluations/**/*.md`의 현재 운영·평가 보고서
- 위 문서를 생성하는 템플릿과 새 문서 작성 계약

`text`, `plaintext`, `mermaid`처럼 사람이 읽는 구조·흐름을 표현하는 펜스형 다이어그램도 문서 본문이다. 실행 가능한 코드 예제와 달리, 그 안의 설명·단계·노드·관계 라벨에는 같은 한국어 계약을 적용한다.

`raw/` 원문, JSON 원장, 실행 코드, 로그, 테스트 fixture와 외부 자료 자체는 번역 대상 문서가 아니다. 다만 원장의 내부 운영 설명이 Wiki에 투영될 때에는 외부 원문이 아닌 설명 문구를 한국어로 작성한다.

## 번역하지 않는 예외

다음 값은 정확성·호환성·출처 추적성을 위해 원형을 유지할 수 있다.

- 코드, 셸 명령, 옵션, 파일명, 경로와 환경 변수
- `CLM-*`, `SRC-*`, `RFC-*` 등 식별자, 해시, 시각, URL과 정확한 locator
- 스키마 키, enum, API·CLI 이름, 프로그래밍 언어 기호와 Skill의 기계 식별자
- 문장 안의 제품명·표준명 같은 고유명사와 추적 가능한 논문 공식 원제. 고유명사를 포함한 일반 제목·설명·문장은 한국어 문맥을 함께 둔다.
- 출처의 공식 원제, 저자·기관명, 정확한 원문 인용과 인용 링크
- OKF가 요구하거나 생산자 확장으로 고정한 `type`, `generated` 등의 기계 값

예외 값이 있는 문서도 그 주변 제목, 설명, 섹션명, 표 머리글과 해설은 한국어여야 한다. 예외는 영문 설명 문단을 그대로 작성할 권한이 아니다.

## 작성 계약

1. 새 문서는 처음부터 한국어로 작성한다.
2. 외부 원제를 보존할 때에는 필요한 경우 한국어 설명을 바로 곁에 둔다.
3. Agent가 만든 요약·평가·근거 설명·상태 사유는 한국어로 기록한다.
4. 자동 생성 문서는 산출물만 고치지 않고 생성 템플릿을 먼저 고친다.
5. `# Citations` 같은 사람이 읽는 관례 표제는 `# 인용`처럼 한국어로 표시한다. 외부 원제와 URL은 그대로 둔다.
6. 번역으로 의미가 바뀔 위험이 있는 증거 문구는 원문을 보존하고 한국어 설명을 별도로 제공한다.

## 검증 계약

독립 실행 명령은 `python3 tools/wiki.py language-validate`다. 이 검사는 전체 단위 테스트와 통합 릴리스 게이트에도 포함한다.

결정론적 검사기는 다음을 실패로 처리한다.

- 적용 범위 문서에 한국어 사람이 읽는 문구가 전혀 없음
- 예외가 아닌 제목·설명·섹션명이 영어로만 작성됨
- 사람이 읽는 일반 문장이 영어로만 작성됨
- Skill 설명·표시 이름·짧은 설명·기본 프롬프트가 한국어가 아님
- 자동 생성 템플릿이 알려진 영문 고정 라벨을 다시 출력함
- README 복사용 전역 부트스트랩과 실제 전역 부트스트랩의 한국어 계약이 어긋남

검사기는 코드 펜스, 인라인 코드, 링크 목적지, ID, 해시, 경로, enum과 명시된 외부 원제·인용을 오탐하지 않아야 한다. 단순히 파일 안에 한글 한 글자가 있는지만 확인해서는 안 된다.

## 수용 기준

- `AC-KO-001`: 중앙 하네스 명세, Skill 명세, Skill 본문, 예약 프롬프트와 Skill UI가 한국어다.
- `AC-KO-002`: `render` 뒤의 `wiki/index.md`, 대시보드, 하위 색인과 상태 투영 문서의 고정 문구가 한국어다.
- `AC-KO-003`: 사람이 작성한 적용 범위 Markdown에 예외가 아닌 영문 전용 제목·설명·문장이 없다.
- `AC-KO-004`: 코드·명령·ID·URL·해시·스키마 값·외부 원제·정확한 인용은 손상되지 않는다.
- `AC-KO-005`: 한국어 문서 검사기가 독립 실행과 전체 테스트에서 통과한다.
- `AC-KO-006`: README, 저장소 `AGENTS.md`, 전역 loader가 이 명세를 연결하고 하네스 변경 전에 spec-driven TDD를 요구한다.
- `AC-KO-007`: 실패 테스트, 최소 구현, 전체 품질 게이트와 롤백 근거가 연결 RFC에 기록된다.
- `AC-KO-008`: 릴리스 검사는 현재 템플릿으로 파생 뷰를 먼저 다시 만들고, 아직 쓰지 않은 릴리스 보고서 Markdown까지 동일한 언어 검사기로 검증한다.
- `AC-KO-009`: `generated: true`, 자동 생성 주석, 링크나 표 안에 있다는 이유만으로 일반 영문 설명을 포괄 면제하지 않는다. 예외는 셀·링크·locator 단위로만 적용한다.
- `AC-KO-010`: RFC 구현 근거는 구조·OKF·보정·보안·실행환경·회귀·메모리 위생·한국어 문서 게이트가 모두 한 번씩 통과한 릴리스 보고서만 사용할 수 있다.
- `AC-KO-011`: `text`, `plaintext`, `mermaid` 다이어그램의 사람이 읽는 영문 전용 라벨을 거부하되, 실행 코드 펜스와 다이어그램 안의 코드·경로·식별자·스키마 값은 보존한다.
- `AC-KO-012`: 사람이 읽는 frontmatter 필드, 짧은 목록, 일반 표와 링크도 본문과 같은 계약을 적용하며, 영문 원문 인용은 출처와 정확한 locator를 함께 표시한 경우에만 보존한다.

## 실패 동작과 롤백

문서 언어 검사가 실패하면 하네스 릴리스를 통과시키지 않는다. 자동 번역으로 원문·식별자·명령을 덮어쓰지 않고, 위반 위치를 보고해 검토 가능한 수정으로 해결한다. 롤백은 검사기와 템플릿 변경을 되돌리되 원문 자료와 append-only 사건 원장은 변경하지 않는다.
